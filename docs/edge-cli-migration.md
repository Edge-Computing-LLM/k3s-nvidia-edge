# edge-cli Migration

`k3s-nvidia-edge` is now the infrastructure layer for the
`Edge-Computing-LLM` platform. The primary operator-facing CLI is
[`edge-cli`](https://github.com/Edge-Computing-LLM/edge-cli).

## Repository Role

This repository owns:

- Linux + k3s + NVIDIA GPU infrastructure workflows.
- The reusable Go package at `pkg/edgebase`.
- The local Helm wrapper chart at `charts/k3s-nvidia-edge`.
- GPU Operator defaults for k3s/containerd edge systems.
- CUDA validation manifests generated by Go code.

This repository does not own:

- LLM application workloads.
- Ollama/Open WebUI/OpenTelemetry/Grafana/Prometheus installation.
- Organization-wide command routing.

Those responsibilities belong to:

- `edge-cli`: unified Go CLI control plane.
- `llm-observability-stack`: LLMOps and observability Helm layer.

## Preferred Commands

Use `edge-cli` for new operator workflows:

```bash
edge doctor
edge install infra --yes
edge validate infra
edge status
edge install all --yes
```

Use this repository directly when developing or testing the infrastructure
package and Helm wrapper:

```bash
make check
bin/k3s-nvidia-edge doctor
bin/k3s-nvidia-edge charts
bin/k3s-nvidia-edge print-commands
```

## Legacy CLI Status

`cmd/k3s-nvidia-edge` remains in the repository during the migration period. It
is useful for compatibility, package development, and direct debugging of the
base layer.

New documentation, demos, and end-to-end flows should prefer `edge-cli`.

## Public API

Downstream Go code should import only:

```go
import "github.com/Edge-Computing-LLM/k3s-nvidia-edge/pkg/edgebase"
```

Do not import `cmd/...` or any private `internal/...` package.

## Asset Boundary

The migration to `edge-cli` does not remove or replace infrastructure assets in
this repository. Keep these assets here:

- `charts/k3s-nvidia-edge`
- `charts/k3s-nvidia-edge/charts/gpu-operator-v26.3.3.tgz`
- `configs/edge-profile.env.example`
- `pkg/edgebase`
- infrastructure documentation

`edge-cli` calls into equivalent infrastructure workflows and uses this repo as
the source of local chart/profile assets.
